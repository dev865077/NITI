# Claim To SPARK Matrix

This matrix maps public proof claims to concrete SPARK package specifications,
proof procedures, and procedure postconditions. It is the claim-level companion
to [`SPARK_TARGET_INVENTORY.md`](SPARK_TARGET_INVENTORY.md).

Each listed procedure is a SPARK subprogram with a `Post =>` contract in the
referenced `.ads` file. A claim not listed here should be treated as
demonstrated, modeled, assumed, or out of scope according to
[`FORMAL_PROOF_BOUNDARY.md`](FORMAL_PROOF_BOUNDARY.md).

## Outside Formal Scope

| Public statement type | Status |
| --- | --- |
| Real secp256k1, BIP340, or SHA-256 security. | Assumed, not proved by SPARK. |
| Bitcoin transaction serialization, sighash, mempool policy, confirmation, fees, or reorg behavior. | Demonstrated only in harnesses or public artifacts where explicitly linked; not proved by SPARK. |
| Oracle truth, liveness, source quality, nonce operations, or non-equivocation outside the modeled scalar relation. | Assumed or monitored by separate oracle evidence, not proved by SPARK. |
| Wallet state retention, backups, production bilateral transport, and user interface safety. | Out of formal scope. |
| Production Lightning channel deployment, route liquidity, force-close behavior, and PTLC support. | Out of formal scope. |
| Market liquidity, solvency, fair pricing, legal enforceability, regulatory treatment, and user-fund safety. | Out of formal scope. |
