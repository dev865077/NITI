# v0.1 Written Execution Transcript

The v0.1 run can be audited from committed artifacts and deterministic local
commands. The transcript below is the shortest written path through the proof
gate, oracle attestation, parent execution, bridge completion, child funding,
negative cases, and the economic-stress entry point.

## 1. Install

```sh
npm ci
```

Expected result: dependencies install without requiring a Bitcoin node for the
artifact-audit path.

## 2. Verify The Public Evidence Bundle

```sh
npm run test:evidence-bundle -- \
  --bundle docs/evidence/public-signet/public-activation-evidence-bundle.json
```

Expected result:

```text
"ok": true
```

The bundle records:

| Object | Evidence |
| --- | --- |
| Parent funding | Public signet funding transaction. |
| Parent CET | Confirmed parent settlement transaction. |
| Bridge | Confirmed bridge transaction. |
| Child funding | Bridge output used by prepared child spends. |
| Negative path | Wrong scalar rejection and pre-resolution invalidity. |

## 3. Run The Deterministic Local Gate

```sh
npm run v0.1:verify
```

Expected result:

```text
v0.1 verification passed.
```

The gate runs the TypeScript deterministic harness, Ada manifest validation,
SPARK no-`pragma Assume` scan, core cDLC proof targets, Lazy window target, and
Lightning companion target.

## Boundary

The transcript supports the narrow v0.1 claim: a parent oracle scalar completes
the parent settlement and bridge activation for a prepared child path, while
non-corresponding scalars and modeled failure cases fail closed.

It does not prove production wallet safety, production mainnet fee policy,
oracle truth, market liquidity, legal readiness, or solvency of a financial
product.
